# Symptoms Checker – экспертная система для первичной медицинской диагностики

## О проекте

Symptoms Checker — это веб-приложение, которое помогает пользователю на основе выбранных симптомов получить ранжированный список возможных заболеваний. Система предназначена для первичной самодиагностики и не заменяет визит к врачу.

**Предметная область:** медицинская диагностика  
**Тип системы:** продукционная экспертная система с взвешенным прямым цепочечным выводом

---

## Как это работает

Пользователь отмечает симптомы из предложенного списка. Механизм логического вывода сопоставляет их с базой знаний и вычисляет для каждого заболевания коэффициент уверенности (confidence) по формуле:

> Уверенность = (совпавшие симптомы / общее количество симптомов в правиле) × вес правила × 100%

Результат сортируется по убыванию уверенности. Пользователь видит только те диагнозы, которые превысили заданный порог.

---

## Структура базы знаний

База знаний представлена в файле `knowledge_base.py`. Каждое заболевание описывается правилом в виде словаря:

- **condition** – название заболевания
- **symptoms** – список характерных симптомов
- **recommendations** – рекомендации для пользователя
- **weight** – вес правила (доверие эксперта к данной корреляции)

Пример правила:

```python
{
    "condition": "Influenza (Flu)",
    "symptoms": ["fever", "cough", "fatigue", "muscle_pain", "sore_throat"],
    "recommendations": "Rest, hydration, and over-the-counter pain relievers.",
    "weight": 0.9
}
```

## Механизм логического вывода
Файл inference_engine.py реализует взвешенный прямой цепочечный вывод:

1. Принимает список симптомов пользователя

2. Для каждого правила вычисляет количество совпадений

3. Рассчитывает уверенность с учётом веса правила

4. Возвращает отсортированный список диагнозов

5. Оболочкой выступает сам Python – система не использует классические оболочки типа CLIPS или Jess, но полностью реализует их ключевой механизм (продукционная модель, прямой вывод, работа с весами уверенности).

## Интерфейс пользователя
Приложение построено на Streamlit и имеет двухпанельную структуру:

* Левая панель (ввод данных)

* Группированные чекбоксы симптомов

* Ползунок порога уверенности (0–100%)

* Кнопка запуска диагностики

Основная область (вывод результатов)

* Ранжированный список заболеваний с индикаторами уверенности

* Рекомендации по каждому диагнозу

* Объяснение: какие симптомы совпали для каждого заболевания

Интерфейс интуитивно понятен и не требует специальной подготовки.

## Тестирование

Система протестирована на следующих сценариях:

| Сценарий | Входные симптомы | Ожидаемый результат | Результат |
|----------|------------------|---------------------|-----------|
| Грипп | fever, cough, fatigue, muscle_pain | Influenza (уверенность > 70%) | ✅ Пройден |
| Простуда | runny_nose, sneezing, sore_throat | Common Cold (уверенность > 80%) | ✅ Пройден |
| COVID-19 | fever, cough, fatigue, loss_of_smell | COVID-19 (уверенность > 75%) | ✅ Пройден |
| Аллергия | sneezing, itchy_eyes, nasal_congestion | Allergy (уверенность > 85%) | ✅ Пройден |
| Пневмония | fever, cough, shortness_of_breath, chest_pain | Pneumonia (уверенность > 60%) | ✅ Пройден |
| Пустой ввод | нет симптомов | ни одного диагноза | ✅ Пройден |

**Выявленные особенности:** система чувствительна к порогу уверенности – при значении ниже 20% могут отображаться маловероятные диагнозы. Рекомендуемое значение порога: 30–40%.

## Запуск проекта

```bash
# Клонирование репозитория
git clone https://github.com/Assem-ElQersh/Expert-Systems
cd Expert-Systems/symptoms-checker

# Установка зависимостей
pip install -r requirements.txt

# Запуск приложения
streamlit run app.py
``` 

После запуска приложение будет доступно по адресу http://localhost:8501
